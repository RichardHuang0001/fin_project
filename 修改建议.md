# 任务目标：扩展 fin_data_svr 项目以支持港股和美股

当前项目是一个基于 MCP (Model Context Protocol) 的 A 股金融数据获取服务，底层强依赖 `baostock`。
现在的目标是：在**不破坏现有 A 股功能**的前提下，引入 `yfinance` 和 `akshare`，将查询范围扩展到港股和美股。

**【核心约束】**
必须完全保留 `src/baostock_data_source.py` 和 `src/tools/news_crawler.py` 中关于新闻爬虫与 AI 分析 (`crawl_news`) 的所有现有逻辑，对该功能不做任何修改。

请严格按照以下 5 个步骤逐步修改代码：

## 第一步：更新依赖配置
请在环境配置文件中引入新的库：
1. 修改 `requirements.txt`，在末尾追加：
   ```text
   yfinance>=0.2.30
   akshare>=1.12.0
   ```
2. 修改 `environment.yml`，在 `pip:` 列表下增加 `- yfinance` 和 `- akshare`。

## 第二步：创建 YFinance 数据源实现
在 `src/` 目录下新建文件 `yfinance_data_source.py`。
1. 导入 `yfinance`，让该类继承自 `src.data_source_interface.FinancialDataSource`。
2. 实现 `get_historical_k_data` 方法：
   - 使用 `yf.download()` 或 `yf.Ticker().history()` 获取美股/港股数据。
   - **重点**：将返回的 DataFrame 列名重命名为全小写，以对齐现有 `baostock` 的格式（如：`Date` -> `date`, `Open` -> `open`, `High` -> `high`, `Low` -> `low`, `Close` -> `close`, `Volume` -> `volume`）。并将 Date 索引重置为普通列并格式化为 `YYYY-MM-DD` 字符串。
3. 实现 `get_stock_basic_info` 方法：
   - 使用 `yf.Ticker(code).info` 获取基本信息，并将其转化为单行的 DataFrame，确保至少包含 `code` 和 `code_name`（可以使用 `shortName` 或 `longName` 映射）。
4. 对于不支持的接口（如 A 股特有的宏观数据、特定指数成分股等），可以抛出 `NoDataFoundError` 或返回空的 DataFrame。

## 第三步：创建多市场路由中心 (核心步骤)
在 `src/` 目录下新建文件 `multi_market_data_source.py`。
1. 创建类 `MultiMarketDataSource`，同样继承自 `FinancialDataSource`。
2. 在 `__init__` 方法中，实例化原有的和新的数据源：
   ```python
   from .baostock_data_source import BaostockDataSource
   from .yfinance_data_source import YFinanceDataSource
   
   class MultiMarketDataSource(FinancialDataSource):
       def __init__(self):
           self.baostock_source = BaostockDataSource()
           self.yfinance_source = YFinanceDataSource()
   ```
3. 实现一个路由判断方法 `_get_source(self, code: str)`：
   - 如果 `code` 以 `sh.` 或 `sz.` 开头，返回 `self.baostock_source`。
   - 否则（美股如 AAPL，港股如 0700.HK），返回 `self.yfinance_source`。
4. **重写 FinancialDataSource 中的所有抽象方法**，在方法内部调用路由到的具体数据源。例如：
   ```python
   def get_historical_k_data(self, code: str, *args, **kwargs):
       source = self._get_source(code)
       return source.get_historical_k_data(code, *args, **kwargs)
   ```
5. **【关键】暴露并透传 crawl_news 方法**：
   因为 `crawl_news` 不在基类接口中，但工具层需要调用。请显式定义该方法，并**无条件路由给 baostock_source**：
   ```python
   def crawl_news(self, query: str, top_k: int = 10) -> str:
       # 始终使用 baostock_data_source.py 中已有的爬虫和 AI 分析逻辑
       return self.baostock_source.crawl_news(query, top_k)
   ```

## 第四步：修改入口文件接入路由
修改 `mcp_server.py`：
1. 导入刚刚写好的路由类：
   ```python
   from src.multi_market_data_source import MultiMarketDataSource
   ```
2. 找到实例化数据源的地方，将其替换为路由类：
   - 修改前：`active_data_source: FinancialDataSource = BaostockDataSource()`
   - 修改后：`active_data_source: FinancialDataSource = MultiMarketDataSource()`
